# ordinance-kr 자치법규 저장소 탐색

대한민국 자치법규를 Git으로 관리하는 저장소다. **각 자치법규 = `본문.md` 파일**, 지방자치단체·자치법규 종류·자치법규명 기준으로 배치된다. 데이터 출처는 [국가법령정보센터 OpenAPI](https://open.law.go.kr).

## 저장소 구조

```
{광역}/
  {기초 또는 _본청 또는 _교육청}/
    {자치법규종류}/
      {자치법규명}/
        본문.md
      {자치법규명}_{공포번호 또는 자치법규ID}/   # 충돌 해소
        본문.md
```

- **광역**: `서울특별시`, `경기도`, `부산광역시` 등
- **기초 슬롯**: 기초자치단체명(예: `수원시`) 또는 광역 본청 `_본청` 또는 `_교육청`
- **자치법규종류**: `조례` `규칙` `훈령` `예규` `고시` `의회규칙` (`기타`는 분류 불명확이라 본문 트리에서 제외)
- 예:
  - `서울특별시/_본청/조례/서울특별시 테스트 조례/본문.md`
  - `경기도/수원시/규칙/수원시 사무전결 처리 규칙/본문.md`
  - `부산광역시/_교육청/조례/부산광역시교육청 교육복지 조례/본문.md`

## 자주 쓰는 작업

저장소 루트(`ordinance-kr/`)에서 실행한다.

```bash
# 서울특별시 본청 조례 보기
ls 서울특별시/_본청/조례/

# 특정 기초자치단체 규칙 보기
ls 경기도/수원시/규칙/

# 특정 자치법규 본문 보기
cat "경기도/수원시/규칙/수원시 사무전결 처리 규칙/본문.md"

# 전체에서 단어 검색 (파일 목록)
grep -rl "공공시설" .
```

## 메타데이터 (YAML Frontmatter)

| 필드 | 설명 |
|------|------|
| `자치법규ID` / `자치법규일련번호` | 고유 식별자 (**안정적 참조 키**) |
| `자치법규명` | 자치법규명 |
| `자치법규종류` | 조례 / 규칙 / 훈령 / 예규 / 고시 / 의회규칙 |
| `지자체기관명` | 지자체명 |
| `지자체구분` | `광역` / `기초` 매핑 (예: `광역: 서울특별시`, `기초: _본청`) |
| `공포일자` / `공포번호` | 공포 정보 |
| `시행일자` | 시행일 |
| `제개정구분` | 제정 / 일부개정 등 |
| `담당부서` | 담당 부서 |
| `본문출처` | `api-text` 또는 `parsing-failed`(조문·부칙 본문 없음, 별표·첨부만) |
| `출처` | law.go.kr URL |
| `공포일자보정` / `공포일자원문` | 공포일자 보정 여부와 원문 값 |

## 반드시 주의할 점

1. **commit hash는 안정 식별자가 아니다.** 파이프라인 개선 시 전체 재구성 후 **force-push**(모든 hash 변경). 장기 참조는 hash 대신 **`자치법규ID`/`자치법규일련번호` + `공포일자` + law.go.kr URL**로. 동기화: `git fetch --all && git reset --hard origin/main`.

2. **`공포일자` 신뢰**: 보정된 경우 `공포일자보정: true`이고 원문은 `공포일자원문`에 보존된다. Git 날짜보다 frontmatter 값을 본다.

3. **`본문출처: parsing-failed`** 문서는 원천 XML에 본문 조문/부칙이 없는 경우(별표·첨부파일만 제공되거나 본문이 비어 있음).

4. **`기타` 종류 제외**: 원천 분류가 불명확한 `기타`는 본문 트리에 포함되지 않는다.
